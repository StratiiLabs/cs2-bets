# cs2-bets

Інструменти для пошуку value-ставок на CS2 Major (Bo3) через API bo3.gg.

## Робочий процес (з WSL, workdir `/mnt/c/Users/stratiiv/projects/cs2-bets`)

```bash
# 1. Завантажити свіжі коефіцієнти (tier S/A, майбутні матчі)
python3 scripts/fetch_odds.py
# або конкретні матчі:
python3 scripts/fetch_odds.py --slug vitality-vs-9z-12-06-2026,aurora-gaming-vs-spirit-12-06-2026

# 2. Записати модельні ймовірності (P перемоги team1 у серії) у predictions.json
#    {"<slug>": 0.80, ...}

# 3. Порахувати EV по всіх ринках + експрес-кандидати
python3 scripts/ev_calc.py data/odds_<stamp>.json --probs predictions.json

# 4. Залогувати ставку (з ключами для CLV!):
python3 scripts/log_bet.py add --event "IEM Cologne Major 2026" \
    --team1 BetBoom --team2 FURIA --market moneyline \
    --selection "BetBoom ML" --coeff 2.44 --stake 8 --model-prob 0.48 \
    --match-slug betboom-team-vs-furia-13-06-2026 --market-key team_1_ml

# 5. ЗА 5-15 ХВ ДО ЛОКУ — зафіксувати закривальну лінію (CLV):
python3 scripts/clv.py capture

# 6. Після матчу — внести результат і дивитись вердикт:
python3 scripts/log_bet.py settle --bet-id <id> --status won|lost
python3 scripts/clv.py report
python3 scripts/log_bet.py stats
```

## CLV — головний детектор edge

CLV (Closing Line Value) = `наш_коеф / закривальний_коеф - 1`.
Закривальна лінія — найгостріша оцінка ринку. Якщо ми СТАБІЛЬНО беремо
вищий коеф, ніж він стає перед локом — у нас реальна перевага, **незалежно
від того, виграла окрема ставка чи ні**. Якщо CLV в нулі/мінусі —
виграші це везіння, і банк на дистанції падатиме.

- Потрібно ~30-50 ставок, щоб вердикт був надійним.
- `clv.py capture` оновлює лінію, поки матч `upcoming`; після старту фіксує
  останнє значення. Тому запускати треба максимально близько до локу.
- `market_key`: `team_1_ml`, `team_2_ml`,
  `team_X_handicap_over_1_5` (+1.5), `team_X_handicap_under_1_5` (-1.5),
  `total_maps_over_2_5`, `score_2_0` тощо (повний список — у знімку odds).

## Файли

- `scripts/fetch_odds.py` — тягне матчі та всі ринки (ML, рахунок, тотали,
  фори) з api.bo3.gg, зберігає знімок у `data/odds_<timestamp>.json`.
- `scripts/fetch_stats.py` — ранг команди + форма за 10 матчів з bo3.gg.
- `scripts/elo.py` — опонент-адаптований Elo (загальний + покартковий). Сирий
  вінрейт оманливий (70% проти слабких < 50% проти топів); Elo це враховує.
  `build` будує рейтинги, `predict --slug` дає ймовірності.
- `scripts/map_stats.py` — покарткові вінрейти команд (з поправкою на силу
  суперника читати обережно — для відносної сили карт усередині команди).
- `scripts/hltv.py` — скрапер HLTV через curl_cffi (обхід Cloudflare без
  браузера). Дістає МАП-ВЕТО (які карти реально граються), склади, H2H зі
  сторінок матчів. Розділ /stats HLTV закритий активним JS-челенджем —
  потрібен браузер (Playwright/FlareSolverr); для карт використовуємо вето.
  Потребує venv із curl_cffi (див. requirements.txt).
- `scripts/today_stats.py` — ранг+форма для всіх команд останнього знімка.
- `scripts/ev_calc.py` — з серійної ймовірності виводить покарткові
  ймовірності (P(2-0), P(2-1)...), рахує EV кожного ринку, чверть-Келлі
  ставки та найкращі експреси з value-ніг різних матчів.
- `scripts/log_bet.py` — лог ставок у bets.csv (add/settle/stats) + ROI + CLV.
- `scripts/clv.py` — захоплення закривальної лінії та звіт по CLV.
- `predictions.json` — модельні ймовірності перемоги team1 по слагах.
- `bets.csv` — журнал ставок з ROI та CLV.
- `GLOSSARY.md` — шпаргалка термінів.
- `data/` — знімки коефіцієнтів і сирі відповіді API.

## Правила (довга дистанція)

1. Ставка лише коли `P(модель) × коеф > 1.05`.
2. Live-матчі (status=current) виключаються автоматично — live-коефіцієнти
   не можна порівнювати з прематчевою моделлю.
3. Розмір ставки: чверть-Келлі, експреси ≤ 0.5% банку.
4. Експрес — тільки з ніг різних матчів, кожна з перевіреним плюсовим EV.
5. Якщо всі value-ноги «в один бік» (всі андердоги +1.5) — це одне модельне
   припущення, а не незалежні переваги. Це сигнал недовіри до моделі,
   а не сім окремих ставок.
6. Величезний EV (+50%+) проти ліквідного ринку Мейджора — найчастіше
   міскалібрація моделі, а не подарунок.

## Відомі обмеження

- HLTV та Liquipedia за Cloudflare — вето/покарткова статистика недоступна,
  покарткові ймовірності виводяться математично з серійної.
- bo3.gg дає лінію одного букмекера (provider 39) — без порівняння ліній.
